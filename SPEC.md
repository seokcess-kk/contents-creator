# 레퍼런스 분석 기반 콘텐츠 생성 엔진 — 통합 기획서 v1.0

> 작성일: 2026-03-31
> 용도: 내부 사용 (수익화 배제)
> MVP 영역: 네이버 블로그 SEO 원고 + 브랜드 이미지
> 초기 업종: 의료(한의원, 피부과, 치과) · 뷰티 · 건강
> 개발 환경: Claude Code + Harness 부트스트랩

---

## 1. 프로덕트 개요

### 한 줄 소개

네이버 상위 노출 콘텐츠의 텍스트·비주얼 패턴을 분석하고, 증명된 패턴 기반으로 클라이언트별 차별화된 블로그 콘텐츠를 생성하는 내부 도구.

### 핵심 차별점

기존 AI 글쓰기 도구(알파블로그, 가제트AI, 뤼튼 등)는 "키워드 → AI 생성" 구조.
이 도구는 **"레퍼런스 분석 → 패턴 추출 → 패턴 기반 생성"** 구조로, AI가 추측이 아닌 데이터를 기반으로 콘텐츠를 만든다.

### 포지셔닝

"AI 글쓰기 도구"가 아닌 **"레퍼런스 분석 기반 콘텐츠 생성"**이라는 새로운 카테고리.
분석에서 가져오는 것(제목 패턴, 키워드 배치, 색상·디자인 경향, 도입부 훅)과 가져오지 않는 것(게시물 전체 구조, 이미지-텍스트 비중)을 구분하여, 데이터 기반이면서도 자체 구조를 유지한다.

### 해결하는 문제

- 동일한 키워드·브랜드를 넣어도 같은 구조의 아웃풋이 생성되어 같은 생산자로 보이는 문제
- 콘텐츠 양산 시 품질 불균일
- 의료 업종 콘텐츠의 의료광고법 위반 리스크
- 레퍼런스 수집·분석이 수작업인 비효율

## 2. 분석 파이프라인

### 2-1. 파이프라인 구조

```
키워드 입력
    ↓
네이버 상위 N개 크롤링
    ↓
┌─ 텍스트 분석 (L1+L2) ─────────┐
│  L1 구조: HTML 파싱              │
│  L2 카피: LLM 분석               │
│                                  │
├─ 비주얼 분석 ──────────────────┤
│  DOM 파싱: 색상 코드, 이미지 수   │
│  VLM 분석: 레이아웃, 색감, 스타일 │
└──────────────────────────────┘
    ↓
키워드별 패턴 카드 생성 (텍스트 + 비주얼 통합)
```

### 2-2. 텍스트 분석 상세

**L1 구조 분석 (코드 기반, LLM 불필요)**
- 글 길이 (총 글자수, 문단 수)
- 섹션 구성 패턴 (도입-본론-결론 비율)
- 소제목(H2/H3) 패턴 및 개수
- 이미지 삽입 위치·빈도·유형
- CTA 배치 위치·문구 패턴
- 네이버 특화 요소 (지도, 구분선, 인용구 블록)

**L2 카피/메시지 분석 (LLM 필요)**
- 제목 패턴 분류 (질문형/숫자형/감정형/방법론형)
- 도입부 훅 패턴 (공감형/통계형/질문형/스토리형)
- 키워드 배치 전략 (제목/첫 문단/소제목/마지막 문단)
- 톤앤매너 분류 (전문가형/친근형/스토리텔링형)
- 설득 구조 분석 (AIDA/PAS/문제-원인-솔루션)
- 연관 키워드·LSI 키워드 추출

### 2-3. 비주얼 분석 상세

**MVP 추출 요소 3가지:**

| 요소 | DOM 파싱 | VLM 분석 |
|------|----------|----------|
| 레이아웃 | 섹션 수, HTML 구조, 이미지/텍스트 면적 비율 | 전체 구성 패턴, 시각적 흐름 |
| 색상 | 정확한 hex 코드, 배경색, 폰트 색상 | 전체 색감 분위기, 업종별 경향 |
| 이미지 소재 | 이미지 태그 수, 위치, 크기 | 유형 분류(실사/AI/일러스트), 오버레이 패턴 |

**분석 방식:** DOM 파싱과 VLM 동등하게 활용.
스크린샷 캡처(Playwright) → VLM에 전달하여 정성 판단.

### 2-4. 패턴 카드

**MVP 단위:** 키워드 단위 (검색어마다 분석)
**Phase 2:** + 업종 단위 집계 (키워드별 분석 결과를 업종 전체로 통합)

패턴 카드 구조:
```
패턴 카드: "{키워드}"
─────────────────────
[텍스트 패턴]
• 평균 글자수, 소제목 수
• 제목 공식 (상위 N개 공통 패턴)
• 도입부 훅 유형 분포
• 설득 구조 유형
• 핵심 연관키워드 세트

[비주얼 패턴]
• 색상 팔레트 경향
• 레이아웃 구성 패턴
• 이미지 유형·배치 경향
```

### 2-5. Phase별 범위

| 구분 | MVP | Phase 2 |
|------|-----|---------|
| L1 구조 분석 | ✅ | 고도화 |
| L2 카피 분석 | ✅ | 고도화 |
| L3 퍼포먼스 연결 | ❌ | 간소 버전 (순위+댓글·공감) |
| 비주얼 분석 | 레이아웃+색상+이미지 소재 | + 타이포그래피, CTA |
| 패턴 카드 단위 | 키워드 단위 | + 업종 단위 집계 |
| 레퍼런스 수집 | 자동 크롤링만 | + 유저 업로드 |

## 3. 클라이언트 프로필

### 3-1. 프로필 구조

**레벨 1: 기본 정보 (필수)**
- 업체명, 대표자/원장명
- 업종·세부 카테고리
- 지역 (시/구 단위)
- 주요 서비스/시술 목록

**레벨 2: 콘텐츠 방향성 (핵심 차별화)**
- 브랜드 톤앤매너 (전문가형/친근형/스토리텔링형 등)
- 타겟 고객 페르소나 (연령, 성별, 고민)
- USP — 한 줄 핵심 차별점
- 자주 쓰는 표현·용어
- 금지 표현·키워드 (법적 이슈 or 브랜드 정책)

**레벨 3: 심화 정보 (Phase 2)**
- 대표 시술/서비스별 상세 설명
- 기존 고성과 콘텐츠 URL
- 경쟁 업체 정보
- 계절·시기별 주력 키워드

### 3-2. 프로필 수집 방식

**MVP:** URL 입력 → 80% 자동 추출 → 20% 수동 확인·보완

```
홈페이지/블로그 URL 입력
    ↓
크롤링 (메인 + 소개 + 서비스 페이지 / 블로그 최근 10편)
    ↓
LLM 분석 → 프로필 초안 자동 생성
    ↓
사용자가 리뷰 & 수정 (금지 키워드 등 수동 보완)
    ↓
프로필 확정 → Supabase 저장
```

자동 추출 가능: 업체명, 원장명, 서비스 목록, 지역, USP 후보, 톤앤매너, 자주 쓰는 표현
수동 입력 필요: 금지 표현·키워드 (콘텐츠에 "없는 것"이라 추출 불가)

## 4. 콘텐츠 생성

### 4-1. 최종 아웃풋 구조 — 패턴 C (하이브리드)

```
[디자인된 헤더 이미지]        ← PNG (후킹·브랜드 인상)
[텍스트: 도입부]              ← 네이버 에디터 텍스트
[실사 사진 위치 표시]          ← MVP: 사용자 수동 삽입
[텍스트: 문제 제기·원인]      ← 네이버 에디터 텍스트
[AI 이미지: 관련 비주얼]       ← 이미지 생성 API
[텍스트: 솔루션·차별점]       ← 네이버 에디터 텍스트
[실사 사진 위치 표시]          ← MVP: 사용자 수동 삽입
[텍스트: 후기·사례]           ← 네이버 에디터 텍스트
[디자인된 CTA 이미지]         ← PNG (전화·예약 유도)
[Disclaimer]
```

**이미지 소재 3종류:**

| 유형 | 역할 | 비중 | 생성 방식 |
|------|------|------|----------|
| 디자인 카드 | 후킹 헤더, CTA, 핵심 메시지 | ★ 핵심 | HTML → PNG 렌더링 |
| 실사 사진 | 신뢰감 (원장, 시술실, 전후) | 보조 | 클라이언트 제공 (MVP: 위치 표시만) |
| AI 이미지 | 분위기 보조 비주얼 | 보조 | 이미지 생성 API |

### 4-2. 변이 엔진 — 5개 층위

"같은 생산자 느낌" 문제를 근본적으로 해결하기 위해 5개 층위 전부 MVP에서 구현.

| 층위 | 구현 방식 | 효과 |
|------|----------|------|
| ① 구조 변이 | 기본 템플릿 풀 5~10개 + 상위글 분석으로 추가 학습 | 매번 다른 섹션 구성 |
| ② 도입부 변이 | 사연형/통계형/질문형/계절형 등 도입 스타일 풀 | 첫 문단 분위기 차별화 |
| ③ 소제목 변이 | 뻔한 소제목 패턴 회피, 다양한 표현 | "~의 장점" 류 반복 방지 |
| ④ 문장 표현 변이 | AI 상투 표현 자동 감지·회피 | "~하는 것이 중요합니다" 류 제거 |
| ⑤ 이미지 배치 변이 | 삽입 위치·개수·유형 변이 | 시각적 차별화 |

**패턴 카드가 제어하는 것 (뼈대):**
- 섹션 구성 순서, 글자수 범위, 소제목 개수
- 제목 공식 유형, 필수 포함 키워드 세트

**패턴 카드가 제어하지 않는 것 (살):**
- 구체적 문장·표현·어휘 선택
- 도입부의 실제 스토리/에피소드
- 사례·후기의 디테일

**변이 적용 방식:** AI가 키워드·클라이언트 프로필 기반으로 변이 조합을 추천 → 사용자 승인 후 생성

### 4-3. 분석 데이터 활용 원칙 (접근 B)

**분석에서 가져오는 것:**
- 효과적인 제목 패턴
- 키워드 배치 전략
- 색상·디자인 경향
- 도입부 훅 패턴
- 연관 키워드 세트

**분석에서 가져오지 않는 것 (자체 결정):**
- 게시물 전체 구조 (패턴 C 하이브리드 고정)
- 이미지-텍스트 비중
- 디자인 카드 스타일

### 4-4. 비주얼 콘텐츠 엔진 — 확장 경로

공통 코어(HTML 생성 → PNG 렌더링 + 변이 시스템)를 공유하며 확장:

```
MVP:     블로그 브랜드 이미지 (680px 섹션별 PNG)
Phase 2: 랜딩페이지 (풀 HTML or 스크린샷)
Phase 3: 상세페이지 (모바일 최적화 긴 스크롤)
```

## 5. 의료광고법 컴플라이언스

### 5-1. MVP 필수 적용

병의원 콘텐츠 특성상 의료법 준수는 MVP부터 무조건 적용.
MediChecker와 blog-automation 두 프로젝트의 학습을 반영하여 새로 설계.

### 5-2. 3중 방어 구조

```
[1차 방어] 생성 시 — 프롬프트에 의료법 규칙 주입
    ↓
[2차 방어] 생성 후 — 자동 검증 (규칙 기반 + LLM 판단)
    ↓
[3차 방어] 위반 발견 시 — 자동 수정 후 재검증
```

### 5-3. 검증 범위

| 콘텐츠 요소 | 검증 필요 | 비고 |
|---|---|---|
| SEO 텍스트 본문 | ✅ 필수 | 과대광고, 비교광고, 보장 표현 등 |
| 디자인 카드 내 텍스트 | ✅ 필수 | 이미지 안에 있어도 텍스트는 검증 대상 |
| 제목 | ✅ 필수 | "100% 완치" 같은 표현 차단 |
| AI 생성 이미지 프롬프트 | ⚠️ 간접 | 전후 사진 연출 등 주의 |

### 5-4. 법적 근거

의료법 제56조 (의료광고의 금지), 제57조 (벌칙), 시행령 제23조 기반.
8개 위반 카테고리 체계 구축 (기존 두 프로젝트에서 검증된 분류 체계 발전).

## 6. 사용자 플로우

### 6-1. MVP 우선 플로우: 클라이언트 온보딩 + 전략 수립 (Flow B)

```
① 클라이언트 등록
   URL 입력 → 프로필 자동 추출 → 확인·수정 → 저장

② 키워드 분석
   키워드 입력 → 상위글 크롤링 → L1+L2+비주얼 분석

③ 패턴 카드 생성
   분석 결과 리뷰 → 패턴 카드 저장

④ 콘텐츠 생성
   패턴 카드 선택 + 클라이언트 프로필 결합
   → AI 변이 조합 추천 → 사용자 승인
   → 원고 생성 (다중 가능)

⑤ 검증 & 출력
   의료법 자동 검증 → 네이버 에디터 최적화 출력
```

### 6-2. 분석↔생성 UX 원칙

- 분석은 1번, 생성은 N번 — 패턴 카드는 자산으로 저장·재사용
- 같은 패턴 카드로 다른 키워드의 원고를 다중 생성 가능
- 클라이언트 프로필이 변이의 핵심 축 (같은 패턴 + 다른 프로필 = 다른 글)

### 6-3. 편집 경험

네이버 에디터에 바로 붙여넣기 최적화.
MVP에서 실사 사진은 위치 표시만 하고 사용자가 수동 삽입.

## 7. 기술 아키텍처

### 7-1. 개발 환경

- **개발 도구:** Claude Code
- **인프라 부트스트랩:** Harness (에이전트팀 + 스킬 자동 생성 → 커스터마이징)
- **방법론:** SDD (Spec Driven Development) + DDD (Domain Driven Design)
- **바이브 코딩 v2.0 가이드라인** 적용

### 7-2. 시스템 아키텍처

Python CLI 파이프라인 중심. 프론트엔드 없이 내부 도구로 운용.

### 7-5. Harness 적용 워크플로우

```
Phase 0: Harness로 에이전트팀 + 스킬 구조 부트스트랩
    ↓   "블로그 콘텐츠 생성 파이프라인 하네스를 구성해줘"
Phase 1: 생성된 구조 리뷰 → 커스터마이징
    ↓   의료법, 변이 엔진 등 도메인 특화 보완
Phase 2: domain/ 코드 실제 구현 (DDD 패턴, Claude Code로)
    ↓
Phase 3: 통합 테스트 + 파이프라인 검증
```

**Harness 아키텍처 패턴 매핑:**

| 파이프라인 구간 | Harness 패턴 | 이유 |
|---|---|---|
| 전체 흐름 | 파이프라인 | 순차 의존 작업 |
| 분석 단계 (L1+L2+비주얼) | 팬아웃/팬인 | 병렬 독립 → 합류 |
| 생성→의료법 검증 | 생성-검증 | 생성 후 품질 검수 |
| 변이 엔진 | 전문가 풀 | 상황별 다른 변이 조합 |

### 7-7. LLM API가 필요한 영역

**LLM 불필요 (코드로 해결):**
- 네이버 상위글 크롤링 (Scrapling + 네이버 API)
- L1 구조 분석 (HTML 파싱)
- 비주얼 분석 DOM 파싱 (CSS/HTML)
- 스크린샷 캡처 (Playwright)
- HTML → PNG 렌더링 (Playwright)
- SEO 검증 정량 체크 (규칙 기반)

**LLM 필요 — 텍스트 분석/생성:**

| 단계 | 역할 | 핵심 요구사항 |
|------|------|-------------|
| L2 카피 분석 | 제목·훅·설득구조·톤 분류 | 긴 컨텍스트, 분석 정확도 |
| 프로필 추출 | 홈페이지에서 USP·톤·서비스 추출 | 비정형→정형 변환 |
| SEO 텍스트 생성 | 패턴+프로필 기반 원고 작성 | 한국어 품질 최우선 |
| 변이 조합 추천 | 최적 변이 파라미터 제안 | 짧은 판단 |
| 의료법 검증 | 위반 여부 판단 + 수정안 | 정밀도 최우선 |
| 디자인 카드 텍스트 | 후킹 카피, CTA 문구 | 임팩트 |

**VLM 필요:**

| 단계 | 역할 |
|------|------|

**이미지 생성 API 필요:**

## 8. 개발 로드맵

### Phase 0: 인프라 부트스트랩
- Harness로 에이전트팀 + 스킬 자동 생성
- 생성된 구조 리뷰 및 커스터마이징
- CLAUDE.md 작성, Supabase 스키마 설계
- dev/active/ 3대 문서 (plan, context, tasks) 초기화

### Phase 1: MVP 핵심 파이프라인
- **1-1:** 크롤러 도메인 (네이버 검색 + 블로그 스크래핑 + 스크린샷)
- **1-2:** 분석 도메인 (L1 구조 + L2 카피 + 비주얼 분석 → 패턴 카드)
- **1-3:** 프로필 도메인 (URL 자동 추출 + Supabase CRUD)
- **1-4:** 의료법 검증 도메인 (3중 방어 구조)
- **1-5:** 생성 도메인 (변이 엔진 + SEO 텍스트 + 디자인 카드)
- **1-6:** 조합 도메인 (최종 HTML 조합 + 렌더링 + 네이버 에디터 출력)

### Phase 2: 고도화
- L3 퍼포먼스 분석 (순위 + 댓글·공감)
- 업종 단위 패턴 카드 집계
- 유저 업로드 레퍼런스
- 클라이언트 프로필 레벨 3
- 실사 사진 라이브러리 관리
- 비주얼 분석 타이포그래피·CTA 추가
- n8n 워크플로우 자동화 연동

### Phase 3: 확장
- 랜딩페이지 생성 (비주얼 콘텐츠 엔진 확장)
- 상세페이지 생성

## 9. 핵심 설계 원칙 요약

| 원칙 | 설명 |
|------|------|
| **레퍼런스는 인사이트, 구조는 자체** | 상위글에서 패턴을 배우되, 게시물 구조는 직접 결정 |
| **변이가 모든 층위에서** | 5개 층위 변이로 "같은 생산자" 느낌 원천 차단 |
| **의료법은 1순위** | 3중 방어, MVP부터 적용, 타협 없음 |
| **분석 1번, 생성 N번** | 패턴 카드와 프로필은 자산으로 재사용 |
| **프로필이 차별화의 핵심** | 같은 패턴 + 다른 프로필 = 다른 글 |
| **품질 먼저, 확장 나중** | 블로그 이미지 품질 완성 후 랜딩/상세 확장 |
