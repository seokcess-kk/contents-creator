# 비주얼 분석 (DOM + VLM)

## DOM 파싱 — 정량 추출

HTML/CSS에서 수치 데이터를 추출한다.

**색상:**
- 인라인 style에서 `color`, `background-color` hex 값 추출
- `<span>`, `<div>` 등의 스타일 속성 파싱
- 빈도 기반 상위 5개 색상을 dominant palette로 선정

**레이아웃:**
- 섹션 수 카운트 (`<div class="se-section-*">`)
- 이미지 태그 수, 위치 (문서 내 상대 위치 %), 크기 (width/height)
- 이미지 영역 vs 텍스트 영역 면적 비율 추정

**이미지 메타:**
- `<img>` 태그 수, src URL, alt 텍스트
- 이미지 크기 속성 (width, height)
- 위치별 분포 (상단/중단/하단)

## VLM 분석 — 정성 판단

스크린샷을 VLM에 전달하여 다음을 분석한다:

| 분석 항목 | VLM 프롬프트 지시 |
|----------|-----------------|
| 전체 색감 | "따뜻한/차가운/전문적/밝은/차분한 중 선택하고 근거 제시" |
| 레이아웃 패턴 | "이미지-텍스트 교차 배치/텍스트 중심/이미지 중심 분류 + 시각적 흐름(시선이 어떤 순서로 이동하는지) 분석" |
| 이미지 스타일 | "실사/AI생성/일러스트/그래픽디자인 중 각 이미지를 분류" |
| 오버레이 패턴 | "텍스트 오버레이, 로고 워터마크, 컬러 필터 유무 판단" |
| 업종 경향 | "이 디자인이 의료/뷰티/건강 중 어느 업종에 최적화되어 보이는지" |

**VLM 프롬프트 원칙:**
- 선택지를 제한하여 일관된 분류 유도
- JSON 출력 강제
- 스크린샷 전체(풀페이지)와 개별 섹션 캡처를 모두 분석한다. 섹션별 캡처는 Playwright의 element screenshot 기능으로 주요 섹션(헤더, 본문 상단, CTA 영역)을 개별 캡처하여 VLM에 전달한다

## DOM-VLM 교차 검증

- DOM 추출 색상과 VLM 판단 색감이 불일치 → DOM hex 값 기준, VLM 분위기 판단은 보조
- DOM에서 이미지 0개인데 VLM이 이미지 있다고 판단 → iframe 내부 이미지 누락 가능성 재확인
- 교차 검증 결과를 `cross_validation` 필드에 기록
