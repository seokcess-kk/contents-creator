# 클라이언트 프로필 자동 추출

## 추출 플로우

```
URL 입력
    ↓
URL 유형 판별 (홈페이지 vs 블로그)
    ↓
크롤링 (메인 + 소개 + 서비스 페이지 / 블로그 최근 10편)
    ↓
LLM 분석 → 프로필 초안 자동 생성
    ↓
사용자 리뷰 & 수정 (금지 키워드 등 수동 보완)
    ↓
프로필 확정 → Supabase 저장
```

## 크롤링 전략

### 홈페이지 URL
1. 메인 페이지 크롤링
2. 메인에서 링크 추출 → 소개, 서비스/시술, 의료진 페이지 자동 탐색
3. 탐색 기준: URL 패턴(/about, /service, /doctor, /introduce) + 네비게이션 메뉴 텍스트 매칭
4. 최대 10페이지까지 크롤링 (과도한 크롤링 방지)

### 블로그 URL
1. 블로그 메인에서 최근 포스트 목록 추출
2. 최근 10편 본문 크롤링
3. 네이버 블로그의 경우 iframe 처리 (naver-crawling 스킬과 동일)

## LLM 추출 프롬프트 구성

각 페이지를 LLM에 전달하여 구조화된 정보를 추출한다.

**추출 지시:**
```
이 웹페이지에서 다음 정보를 추출하라:
1. 업체명 (정확한 상호)
2. 대표자/원장명
3. 업종 및 세부 카테고리
4. 지역 (시/구 단위)
5. 주요 서비스/시술 목록 (각각 한 줄 설명 포함)
6. USP (핵심 차별점) 후보 — 반복적으로 강조하는 메시지가 무엇인지
7. 자주 사용하는 표현/용어

각 항목에 대해:
- 추출한 값
- 신뢰도 (high/medium/low)
- 출처 (어느 페이지의 어느 부분에서 추출했는지)
를 JSON으로 반환하라.
```

**톤앤매너 분류 (블로그 기반):**
블로그 10편 종합 분석 → 전문가형/친근형/스토리텔링형/정보전달형 중 분류

## 프로필 레벨 구조

### 레벨 1: 기본 정보 (필수, 자동 추출 목표)
- 업체명, 대표자/원장명
- 업종·세부 카테고리
- 지역 (시/구 단위)
- 주요 서비스/시술 목록

### 레벨 2: 콘텐츠 방향성 (핵심 차별화, 일부 수동)
- 브랜드 톤앤매너 → 자동 추출
- 타겟 고객 페르소나 → 수동 입력
- USP → 자동 추출 + 사용자 확인
- 자주 쓰는 표현·용어 → 자동 추출
- 금지 표현·키워드 → **반드시 수동 입력** (콘텐츠에 "없는 것"은 추출 불가)

### 레벨 3: 심화 정보 (Phase 2)
- 대표 시술별 상세 설명
- 기존 고성과 콘텐츠 URL
- 경쟁 업체 정보
- 계절별 주력 키워드

## Supabase 저장

프로필 확정 후 Supabase `client_profiles` 테이블에 저장.
- client_id: UUID
- profile_data: JSONB (draft_profile.json 스키마)
- created_at, updated_at: timestamp
- status: "draft" | "confirmed"

프로필은 수정 가능. 수정 시 updated_at 갱신하고, 변경 이력을 `profile_history` 테이블에 기록.
