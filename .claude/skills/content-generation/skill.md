# 콘텐츠 생성 (변이 엔진 + SEO 텍스트 + 디자인 카드)

## 생성 흐름

```
패턴 카드 로드 → 클라이언트 프로필 로드
    ↓
[의료 업종 판정] → 의료법 규칙 주입 준비 (1차 방어)
    ↓
변이 조합 결정 (5개 층위) → 사용자 승인
    ↓
SEO 텍스트 생성 (LLM + 의료법 규칙 주입)
    ↓
디자인 카드 HTML 생성
    ↓
AI 이미지 프롬프트 생성
```

## 의료법 1차 방어 — 생성 시 프롬프트 주입

3중 방어의 첫 번째 계층. 콘텐츠 생성 LLM 호출 시 의료광고법 규칙을 시스템 프롬프트에 주입하여 위반 표현이 애초에 생성되지 않도록 예방한다.

**적용 조건:**
- 클라이언트 업종이 의료(한의원/피부과/치과 등)인 경우: 필수 적용
- 뷰티·건강 업종 + 시술 관련 키워드: 조건부 적용

**주입 내용:**
1. 8개 위반 카테고리 요약 및 구체적 금지 표현 목록 (`references/prohibited-expressions.md`에서 로드)
2. 금지 표현별 허용 대체 표현 사전
3. "치료 보장, 최상급 비교, 전후 사진 연출을 절대 포함하지 마라" 명시 지시
4. Disclaimer 자동 삽입 지시

**주입 위치:** LLM 시스템 프롬프트의 끝부분에 `[의료광고법 준수 규칙]` 섹션으로 삽입

## 변이 조합 사용자 승인

변이 조합을 결정한 후, 생성 전에 사용자에게 조합을 제시하고 승인을 받는다.

**제시 형식:**
```
[변이 조합 제안]
① 구조: 문제제기→원인→솔루션→차별점→사례→CTA
② 도입부: 통계형 ("최근 조사에 따르면...")
③ 소제목 스타일: 질문형 + 구체적 상황
④ 표현 톤: 전문가형 (프로필 기반)
⑤ 이미지 배치: 도입 후 1장 + 솔루션 후 1장 + CTA 전 1장

[승인/수정/재추천] 중 선택해 주세요.
```

사용자가 수정 요청 시 해당 층위만 변경. 재추천 시 전체 조합을 새로 생성.

## 5개 층위 변이 엔진

### ① 구조 변이
기본 템플릿 풀에서 선택. 패턴 카드의 section_order를 기반으로 하되, 순서와 비중을 변이한다.

템플릿 예시:
- `문제제기→원인→솔루션→차별점→사례→CTA`
- `스토리→공감→정보→솔루션→비교→CTA`
- `질문→답변→근거→사례→차별점→CTA`
- `계절이슈→연결→서비스소개→효과→후기→CTA`
- `통계→문제인식→해결방법→전문가팁→CTA`

### ② 도입부 변이
| 스타일 | 예시 패턴 |
|--------|----------|
| 사연형 | "00님은 매일 아침 거울을 보며..." |
| 통계형 | "최근 조사에 따르면 10명 중 7명이..." |
| 질문형 | "혹시 이런 고민을 하고 계신가요?" |
| 계절형 | "날씨가 추워지면서 OO 고민이..." |
| 공감형 | "많은 분들이 OO 때문에 걱정하시죠" |
| 반전형 | "OO가 오히려 해로울 수 있다는 사실, 알고 계셨나요?" |

### ③ 소제목 변이
금지 패턴: "~의 장점", "~란?", "~하는 방법" 의 단순 반복
대체 전략: 질문형, 숫자 포함, 감정 유발, 구체적 상황 묘사

### ④ 문장 표현 변이
**AI 상투 표현 블랙리스트:**
- "~하는 것이 중요합니다"
- "~에 대해 알아보겠습니다"
- "~하는 것을 추천드립니다"
- "그렇다면 ~는 어떨까요?"
- "~라고 할 수 있습니다"

생성 후 블랙리스트 매칭 → 자연스러운 대체 표현으로 교체

### ⑤ 이미지 배치 변이
패턴 카드의 image_count_range 내에서 위치·개수·유형 조합을 변이한다.

**위치 변이 패턴:**
| 패턴 | 배치 | 효과 |
|------|------|------|
| 헤더 집중형 | 헤더 이미지 1장 + 본문 텍스트 위주 | 정보 중심 글에 적합 |
| 균등 분산형 | 2~3문단마다 1장씩 | 가독성·스크롤 유도 |
| 솔루션 강조형 | 문제 섹션 텍스트만 → 솔루션 섹션에 이미지 집중 | 차별점 시각화 |
| 전후 대비형 | 도입부 1장 + 결론부 1장 | 변화 임팩트 (의료 업종 시 전후 사진 규정 준수) |
| CTA 집중형 | 본문 최소 → CTA 주변 이미지 다수 | 전환 유도 |

**유형 배분 변이:**
패턴 카드의 image_types 비율을 기준으로, ±20% 범위에서 유형별 비중을 변이한다.
예: 패턴 카드가 {실사:50%, AI:30%, 디자인카드:20%}이면 → {실사:40%, AI:40%, 디자인카드:20%}도 허용

### 동적 템플릿 학습

패턴 카드의 section_order와 상위글 구조 분석 결과에서 기본 템플릿 풀에 없는 새로운 구조 패턴이 발견되면, 해당 패턴을 구조 변이 풀에 동적 추가한다. 기본 5~10개 + 분석 기반 추가 학습으로 템플릿 풀을 확장한다.

## SEO 텍스트 생성 원칙

- 패턴 카드의 required_keywords를 제목, 첫 문단, 소제목, 마지막 문단에 자연스럽게 배치
- related_keywords를 본문 전체에 분산 배치 (키워드 스터핑 금지)
- 네이버 블로그 SEO: 제목 30자 내외, 본문에 키워드 3~5회, 소제목에 키워드 1~2회
- 클라이언트 프로필의 톤앤매너, USP, 금지 표현을 반영

## 디자인 카드 HTML 생성

패턴 카드의 visual_pattern(색상, 레이아웃)을 반영하여 HTML을 생성한다.
- 헤더 카드: 키워드 + 후킹 카피 + 브랜드 색상
- CTA 카드: 전화번호/예약 유도 + 브랜드 정보
- 크기: 680px 너비 고정 (네이버 블로그 기준)
- 폰트: 웹폰트 또는 시스템 폰트 (렌더링 안정성)

## AI 이미지 프롬프트 생성

- 한글/영문 텍스트 삽입 지시 금지 (이미지 생성 AI의 텍스트 렌더링 품질 불안정)
- 분위기/무드 중심 프롬프트 작성
- 의료 시술 전후 연출 금지 (의료광고법)
