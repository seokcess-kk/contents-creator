# 텍스트 분석 (L1 + L2)

## L1 구조 분석 — 코드 기반, LLM 호출 금지

BeautifulSoup/lxml로 HTML을 파싱하여 정량 데이터를 추출한다.

**추출 항목:**
- 총 글자수 (태그 제거 후 순수 텍스트)
- 문단 수 (`<p>`, `<div class="se-text-paragraph">`)
- 소제목 수와 텍스트 (`<h2>`, `<h3>`, `<strong>` 기반 소제목)
- 섹션 구성 비율 (도입:본론:결론을 글자수 비율로 계산)
- 이미지 위치 (몇 번째 문단 뒤에 삽입되었는지)
- CTA 위치와 문구 ("전화", "상담", "예약", "카카오톡" 등 키워드 탐지)
- 네이버 특화 요소 카운트 (지도 embed, 구분선, 인용구)

**섹션 경계 판단 기준:**
- 도입부: 첫 번째 소제목 이전까지
- 결론: 마지막 CTA 포함 문단부터
- 본론: 나머지

## L2 카피/메시지 분석 — LLM 기반

각 포스트를 LLM에 개별 전달하여 분석한 뒤, N개 결과를 종합한다.

**분석 항목:**

| 항목 | 분류 체계 |
|------|----------|
| 제목 패턴 | 질문형 / 숫자형 / 감정형 / 방법론형 / 리스트형 |
| 도입부 훅 | 공감형 / 통계형 / 질문형 / 스토리형 / 시즌형 |
| 톤앤매너 | 전문가형 / 친근형 / 스토리텔링형 / 정보전달형 |
| 설득 구조 | AIDA / PAS / 문제-원인-솔루션 / 비포-애프터 |
| 키워드 배치 | 제목 / 첫문단 / 소제목 / 마지막문단 / 본문 전체 |

**LLM 프롬프트 구성 원칙:**
- 분류 체계를 프롬프트에 명시하여 일관된 라벨링 유도
- 예시와 함께 분류 기준을 제공
- JSON 출력 강제 (structured output)

## 종합 로직

N개 포스트의 개별 분석 결과를 집계한다:
- 각 분류 항목별 분포(빈도수, 비율) 계산
- 상위 패턴 2~3개를 "주요 패턴"으로 선정
- 대표 예시 문장을 각 패턴에 첨부
