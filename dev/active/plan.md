## Phase 1 실행 순서 (MVP)

SPEC.md 섹션 8의 Phase 1 기준. 도메인 간 의존성을 고려한 구현 순서:

```
1-1. 크롤러 도메인      ← 다른 모든 도메인의 입력 데이터 제공, 최우선
1-2. 분석 도메인        ← 크롤러 출력 소비, 패턴 카드 생산
1-3. 프로필 도메인      ← 크롤러와 독립적, 1-1과 병렬 가능
1-4. 의료법 검증 도메인  ← 생성 전 규칙 정립 필요, 1-5 전에 완료
1-5. 생성 도메인        ← 패턴 카드 + 프로필 + 의료법 규칙 필요
1-6. 조합 도메인        ← 생성 출력 소비, 최후순
```

## 의존성 그래프

```
1-1 크롤러 ──→ 1-2 분석 ──→ 1-5 생성 ──→ 1-6 조합
                            ↗          ↗
1-3 프로필 ────────────────┘          /
1-4 의료법 검증 ─────────────────────┘
```
